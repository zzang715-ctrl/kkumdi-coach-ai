# 최종 QA 체크리스트

이 문서는 꿈디코치 AI 강사비서를 배포하기 전에 직접 눌러 보며 확인할 항목입니다.

## 기본 화면

1. 대시보드가 깨지지 않고 열리는지 확인합니다.
2. 완성도 확인, 설정 점검, 사용 안내 버튼이 이동되는지 확인합니다.
3. 모바일 크기에서도 글자가 한 글자씩 내려가지 않는지 확인합니다.

## 프로젝트

1. 새 프로젝트를 만들고 저장 메시지가 보이는지 확인합니다.
2. 저장된 프로젝트 목록에 방금 만든 프로젝트가 보이는지 확인합니다.
3. 프로젝트 복제와 이 프로젝트 삭제 버튼이 기대대로 동작하는지 확인합니다.

## 8단계 작성 흐름

1. 제안서부터 마케팅까지 각 단계 화면에 들어갈 수 있는지 확인합니다.
2. 샘플 초안 만들기, 복사, 저장 버튼이 동작하는지 확인합니다.
3. 저장 중, 저장됨, 저장 필요 상태가 잘 보이는지 확인합니다.

## 다운로드

1. 다운로드 화면에서 비어 있는 항목과 작성된 항목이 구분되는지 확인합니다.
2. Markdown 저장 버튼이 동작하는지 확인합니다.
3. Word와 PDF 저장 안내가 글자 깨짐 없이 보이는지 확인합니다.

## 설정과 서버 저장

1. OpenAI 결제 전 quota 오류가 앱 문제처럼 보이지 않는지 확인합니다.
2. Supabase 테스트 버튼이 연결 상태를 친절하게 알려주는지 확인합니다.
3. Supabase SQL 복사 버튼이 동작하는지 확인합니다.

## 배포 전 명령어

```powershell
npm.cmd run lint
npm.cmd run build
```

두 명령어가 모두 통과하면 Vercel 배포 준비가 훨씬 안전해집니다.

## QA 결과 저장

앱의 `http://localhost:3000/qa-guide` 화면에서 체크를 진행한 뒤 `QA 결과 내보내기` 버튼을 누르면 Markdown 파일로 점검 결과를 저장할 수 있습니다.

QA 중 발견한 문제는 같은 화면의 `QA 문제 메모`에 적어둘 수 있습니다. 이 메모도 QA 결과 파일에 함께 저장됩니다.
